Описание сценариев:
=====================
* ### Название прецедента:
Посмотреть таблицы с данными.
### Дейсвующие лица:
Преподаватель или неавторизированный пользователь.
### Цель:
Посмотреть интересующую информацию для подгруппы.
### Предусловие:
нет
### Главная последовательность:
1. Пользователь кликает в выпадающем меню по типу таблицы, которую
желает посмотреть (оценки либо посещения);
2. Система отображает таблицу выбранного типа для предыдущей выбранной подгруппы;
3. Пользователь верху страницы кликает в выпадающем меню номера группы по номеру
подгруппы;
4. Система отображает таблицу выбранного типа для выбранной подгруппы.
### Альтернативная последовательность(нет нужных данных):
1. Пользователь кликает в выпадающем меню по типу таблицы, которую
желает посмотреть (оценки либо посещения);
2. Система перенаправит пользователь на страницу, информирующую об ошибке.
---------------
* ### Название прецедента:
Посмотреть информацию о студенте.
### Дейсвующее лицо:
Преподаватель.
### Цель:
Посмотреть инфорацию о студенте.
### Предусловие:
Загрузить таблицу любого типа для подгруппы группы, в которой учится студент.
### Главная последовательность:
1. Пользователь кликает по имени имени и фамилии студента;
2. Система отображает окно с ФИ, почтой, ссылкой на репозиторий студента;
-------
* ### Название прецедента:
Отметить отсутствие студента на занятии.
### Дейсвующее лицо:
Преподаватель.
### Цель:
Запомнить, что студент не пришел на занятие.
### Предусловие:
Загрузить таблицу "Посещение" для подгруппы группы, в которой учится студент.
### Главная последовательность:
1. Пользователь кликает по пересечению строки, в которой записан интересующий студент,
и столбцу, в заголовке которого указана дата занятия;
2. Система окрашивает ячейку в красный цвет и добавляет в нее, букву "н";
3. Система передает на сервер данные;
4. Система записывает в БД отсутствие студента.
### Альтернативная последовательность (студент уже отмечен отсутсвующим):
1. Пользователь кликает по пересечению строки, в которой записан интересующий студент,
и столбцу, в заголовке которого указана дата занятия;
2. Система меняет цвет ячейки из красного цвета в белый и удаляет букву "н";
3. Система передает на сервер данные;
4. Система записывает в БД присутсвие студента.
---------------
* ### Название прецедента:
Оценить студента.
### Дейсвующее лицо:
Преподаватель.
### Цель:
Зафиксировать оценку студента за лабораторную работу, тест или изменить бонус студента.
### Предусловие:
Загрузить таблицу "Оценки" для подгруппы группы, в которой учится студент.
### Главная последовательность:
1. Пользователь кликает по пересечению строки, в которой записан интересующий студент,
и столбцу, в заголовке которого указан интерисующий критерий;
2. Вводит оценку;
3. Система передает на сервер данные;
4. Система записывает в БД оценку.
------------
* ### Название прецедента:
Добавить критерий оценивания.
### Дейсвующее лицо:
Преподаватель.
### Цель:
Визуально добавить колонку в таблицу "Оценки": выдать лаборатоную работу или добавить тест.
### Предусловие:
Загрузить таблицу "Оценки" для подгруппы группы, в которой учится студент.
### Главная последовательность:
1. Пользователь кликает по кнопке "Добавить".
2. Система выводит форму;
3. Пользователь заполняет форму: выбирает тип (лабораторная или тест);
4. Система передает на сервер данные;
5. Система записывает в БД новый критерий оценивания.
----------------
* ### Название прецедента:
Добавить студента.
### Дейсвующее лицо:
Преподаватель.
### Цель:
Добавить нового студента в список подгруппы.
### Предусловие:
Загрузить таблицу "Редактирование" для подгруппы группы, в которой учится студент.
### Главная последовательность:
1. Пользователь кликает по кнопке "Добавить".
2. Система выводит форму;
3. Пользователь вводит имя и фамилию студента (обязательное поле);
4. Пользователь вводит почтовый адрес (необязательное поле);
5. Пользователь вводит ссылку на репозиторий (необязательное поле);
6. Пользователь нажимает кнопку "Сохранить";
7. Система передает на сервер данные;
8. Система записывает в БД студента.
----------------------------
* ### Название прецедента:
Удалить студента.
### Дейсвующее лицо:
Преподаватель.
### Цель:
Удалить студента из списка подгруппы.
### Предусловие:
Загрузить таблицу "Редактирование" для подгруппы группы, в которой учится студент.
### Главная последовательность:
1. Пользователь кликает по имени имени и фамилии студента;
2. Система выводит форму, заполненную информациейо студенте;
3. Пользователь нажимает кнопку "Удалить";
4. Система передает на сервер данные;
5. Система студента из БД.
-----------
* ### Название прецедента:
Изменить информацию о студенте.
### Дейсвующее лицо:
Преподаватель.
### Цель:
Редактирование студента.
### Предусловие:
Загрузить таблицу "Редактирование" для подгруппы группы, в которой учится студент.
### Главная последовательность:
1. Пользователь кликает по имени имени и фамилии студента;
2. Система выводит форму, заполненную информациейо студенте;
3. Пользователь редактирует интересующие поля;
4. Пользователь нажимает кнопку "Сохранить";
5. Система передает на сервер данные;
6. Система заменяет в БД студента.
-------
* ### Название прецедента:
Инициализировать проверку репозиториев.
### Дейсвующее лицо:
Таймер.
### Цель:
Запустить систему проверки репозиториев студентов и высставление студентам коэффициентов за лабораторную работу.
### Предусловие:
нет
### Главная последовательность:
1. Таймер ожидает выставленное время;
2. Таймер запускает систему;
3. Система проверяет репозитории (взаимодействие с gitAPI).
В случае, если коммит был сделан, выставляет коэффициент.
-----------
* ### Название прецедента:
Инициализировать проверку расписания занятий для групп.
### Дейсвующее лицо:
Таймер.
### Цель:
Запустить систему, которая добавляет занятия на текущий день.
### Предусловие:
нет
### Главная последовательность:
1. Таймер ожидает выставленное время;
2. Таймер запускает систему;
3. Система проверяет расписание групп (взаимодействие с scheduleAPI).
В случае, если занятие для группы будет в данный день, то добавляет в список занятий.
